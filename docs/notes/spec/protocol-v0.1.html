unstable-ui 协议草案 v0.1

这一版协议以“先跑通最小闭环”为目标，覆盖当前仓库已经实现或已经明确准备支持的事件和数据结构。后续允许扩展，但不应随意破坏已有语义。

1. 协议总形态

协议是事件驱动的，不是一次性返回整页 HTML 或组件代码。harness 对客户端的主要输出是事件流，客户端对 harness 的主要输入也是事件。

  • harness 负责表达“此刻展示什么、等待什么、正在处理什么”。
  • 客户端负责状态管理、渲染和把用户操作回传成结构化事件。
  • 双方通过 JSON 交互，不传可执行 UI 代码。

2. Harness 事件

事件 作用 当前状态
session.started 声明 session 已创建，并返回 sessionId 已实现
status 更新运行阶段，例如 waiting、thinking、running 已实现
screen.updated 下发新的 screen schema 已实现
artifact.available 声明某个 artifact 可被预览或打开 已实现基础状态
capability.requested 向客户端请求设备能力 协议已定义，桥接未做
error 展示可读错误信息 已实现
session.completed 声明当前任务已结束 已实现基础状态 
{
  "type": "status",
  "phase": "thinking"
}

3. 客户端事件

事件 作用 当前状态
session.start 发起新会话 协议已定义
voice.input 提交转写文本和可选音频地址 协议已定义，语音壳未做
action.triggered 回传用户点击或操作的 action 已实现
artifact.requested 请求 preview / open / share / download 已实现基础状态
capability.resolved 返回设备能力请求结果 协议已定义

4. Block 集合

当前内建 block 只保留最小集合，不追求一开始就完整。

  • text：纯文本块。
  • card：标题 + 正文概览块。
  • list：列表条目集合。
  • actions：一组操作按钮。
  • resource：单个 artifact 资源块。
{
  "id": "actions-1",
  "type": "actions",
  "items": [
    { "id": "show-plan", "label": "Show plan" },
    { "id": "preview-asset", "label": "Preview asset" }
  ]
}

5. Artifact

Artifact 用来统一表达本地或远端产物。当前类型包括 text / image / audio / video / pdf / html / file / link / json。 

{
  "id": "artifact-1",
  "kind": "link",
  "uri": "https://github.com",
  "source": "remote",
  "title": "GitHub",
  "previewable": true,
  "openable": true
}

6. Capability

Capability request 用来请求设备能力。目前协议里已预留 microphone / camera / photo-library / location / file-picker / share / open-url。 

{
  "type": "capability.requested",
  "request": {
    "id": "req_1",
    "capability": "microphone",
    "reason": "需要录入语音输入"
  }
}

7. 当前判断

  • 第一版协议坚持“事件 + 受约束 block schema”，不回退到任意组件代码。
  • artifact 与 capability 现在先在协议层定清楚，桥接能力后补。
  • 如果后续出现更复杂布局需求，优先扩展 block 和 layout schema，而不是引入可执行 UI。
